Welcome to IRIS

The IRIS interaction framework is a new way to define application interaction and intends to makes it easy for developers to create interactive REST serivces, or put another way a service accessed on behalf of a human. We think one of the best ways to provide separation of view and resource manager concerns is to introduce an interaction layer between the user agent and resource manager. In MVC terms we are facilitating rich interactions and great user experience by putting a gateway between the view and controller in the user agent and a model mapped to the resources in the resource managers.

This interaction layer solves a major headache for managers of self assisted channel services, where it is typically very labor intenstive to achieve unified interaction or experience across all channels. To manage interactions across internet, mobile, tablet, voice, or specialist devices, IRIS provides a common access point for the various user agents, mash-up / synchronous call-out capabilities, and a language to describe the Resource Interaction Model.

Our mains goals:

  • to create web based services according to RESTful constraints
  • aggregate / mashup multiple resource managers into a single interaction service
  • program Hypermedia (HATEOAS) interactions with a state machine domain specific language (DSL)
